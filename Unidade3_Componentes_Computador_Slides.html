Aula 1 · Barramentos
Barramentos do Sistema

Barramentos são vias de comunicação que transportam sinais elétricos entre os componentes do computador. São a "estrada" por onde trafegam dados, endereços e sinais de controle.

DADOS

Transporta os dados entre CPU, memória e periféricos. A largura (8, 16, 32, 64 bits) define a quantidade transferida por ciclo.

ENDEREÇO

Indica a posição de memória a ser acessada. A largura define o espaço máximo de endereçamento (ex: 32 bits = 4 GB).

CONTROLE

Sinais que coordenam as operações: leitura, escrita, interrupções, clock e status dos dispositivos.

Aula 1 · Clock e Desempenho
Clock, Frequência e Desempenho

O clock é um oscilador que define o ritmo das operações da CPU. Cada "tick" do clock permite a execução de parte de uma instrução.

Multi-core

Múltiplos núcleos em um chip permitem executar várias tarefas em paralelo. Um i7 quad-core pode processar 4 threads simultaneamente.

FPU (Unid. Ponto Flutuante)

Coprocessador especializado em operações decimais. Essencial para jogos, renderização 3D e computação científica.

Pipeline

Técnica que permite iniciar uma nova instrução a cada ciclo de clock, sem esperar a anterior terminar completamente.

EVOLUÇÃO DO CLOCK

Intel 4004 (1971) 740 KHz
Intel 8086 (1978) 5-10 MHz
Pentium (1993) 60-200 MHz
Core 2 (2006) 1.8-3.0 GHz
Atual (2024+) 3.0-6.0 GHz

Frequência estabilizou ~2005. Ganhos agora vêm de mais núcleos e melhor arquitetura.

Aula 1 · Arquiteturas de Processadores
CISC vs RISC

CISC

Complex Instruction Set

  • Conjunto extenso de instruções complexas
  • Cada instrução pode realizar várias operações
  • Ciclos de instrução mais longos
  • Mais registradores de propósito geral
  • Usado em PCs (Intel, AMD)
  • Domina computadores pessoais e servidores
VS

RISC

Reduced Instruction Set

  • Conjunto reduzido de instruções simples
  • Cada instrução é rápida e otimizada
  • Ciclos de instrução muito curtos
  • Menos registradores, mas eficiente
  • Usado em mobile (ARM)
  • Domina smartphones e tablets
Aula 1 · Ciclo de Instrução
Ciclo de Instrução da CPU
PASSO 1

Busca (Fetch)

A CPU lê a próxima instrução da memória usando o endereço do PC.

→
PASSO 2

Decodificação

A UC interpreta a instrução e identifica a operação a ser realizada.

→
PASSO 3

Execução

A ALU realiza a operação (aritmética, lógica ou transferência de dados).

→
PASSO 4

Armazenamento

O resultado é salvo em registradores ou na memória. O PC é atualizado.

O ciclo se repete continuamente enquanto o processador estiver em operação.

Aula 2 · A Memória na Arquitetura
Memória na Arquitetura Von Neumann

Na arquitetura de Von Neumann, instruções e dados compartilham a mesma memória. A CPU busca instruções e dados pelo mesmo barramento — o chamado gargalo de Von Neumann.

A hierarquia de memória surgiu para mitigar esse gargalo: memórias rápidas e pequenas perto da CPU, lentas e grandes mais distantes.

Aula 2 · Memória Principal
RAM — Random Access Memory

A RAM é a "mesa de trabalho" do computador — onde ficam os dados e programas em execução. É volátil: perde os dados ao desligar.

A DRAM (Dynamic RAM) precisa ser constantemente "refrescada" para manter os dados — mais barata e usada como memória principal.

EVOLUÇÃO DDR

DDR2 Mais velocidade, menos energia
DDR3 Padrão por vários anos
DDR4 Melhor eficiência energética
DDR5 Velocidades superiores — atual
Aula 2 · SRAM vs DRAM
SRAM vs DRAM

SRAM

Static RAM

  • Usa flip-flops (6 transistores por bit)
  • Não precisa de refresh
  • Muito rápida (~1-10 ns)
  • Consome mais energia
  • Custo alto por bit
  • Usada em memória Cache
VS

DRAM

Dynamic RAM

  • Usa capacitores (1 transistor + 1 capacitor)
  • Precisa de refresh constante (ms)
  • Mais lenta (~50-70 ns)
  • Consome menos energia
  • Custo baixo por bit
  • Usada em memória Principal (RAM)

O refresh da DRAM é necessário porque os capacitores perdem carga elétrica com o tempo.

Aula 2 · Memória Cache
Memória Cache — o atalho da CPU

A cache atua como ponte rápida entre o processador e a RAM, armazenando dados frequentemente acessados. Sem ela, a CPU ficaria ociosa esperando a memória principal.

Localidade Temporal

Se um dado foi acessado, provavelmente será acessado novamente em breve.

Localidade Espacial

Se um dado foi acessado, dados vizinhos também serão requisitados.

Aula 2 · Mapeamento de Cache
Técnicas de Mapeamento
D

Mapeamento Direto

Cada bloco da memória principal só pode ocupar uma posição fixa na cache. Simples e rápido, mas gera conflitos.

A

Totalmente Associativo

Qualquer bloco pode ir para qualquer posição da cache. Menos conflitos, porém hardware mais complexo e caro.

C

Conjunto Associativo

Meio-termo: cache dividida em conjuntos e cada bloco pode ir para qualquer posição dentro de um conjunto.

Política de Substituição

Quando a cache está cheia: LRU (Least Recently Used) — remove o bloco menos recentemente acessado.

Coerência de Cache

Em multi-core, garantir que todos os núcleos vejam os mesmos dados atualizados na cache.

Aula 3 · Disco Magnético
Disco Magnético — Estrutura e Tempos

O HDD é composto por pratos magnéticos que giram em alta velocidade (5400-7200 RPM). Uma cabeça de leitura/escrita se move sobre as trilhas concêntricas.

Trilha

Circunferência concêntrica na superfície do disco.

Setor

Menor unidade endereçável (tipicamente 512 bytes).

Cilindro

Conjunto de trilhas na mesma posição em diferentes pratos.

TEMPOS DE ACESSO

Seek Time 3-15 ms

Tempo para mover a cabeça até a trilha correta.

Latência Rotacional 2-6 ms

Tempo para o setor girar até a cabeça.

Transferência < 1 ms

Tempo efetivo de leitura/escrita dos dados.

Tempo total = Seek + Latência + Transferência

Aula 3 · HDD vs SSD
HDD vs SSD

HDD

Disco Magnético

  • Partes móveis (disco + cabeça)
  • Velocidade: até ~160 MB/s
  • Mais suscetível a impactos
  • Custo por GB: muito baixo
  • Vida útil: sem limite de escrita
  • Mais pesado e gera ruído
VS

SSD

Memória Flash

  • Sem partes móveis
  • Velocidade: até 550 MB/s (SATA) ou 5+ GB/s (NVMe)
  • Resistente a impactos
  • Custo por GB: mais alto
  • Ciclos de escrita limitados
  • Leve, silencioso, baixo consumo

Aula 4 · Evolução das Técnicas de E/S
Técnicas de gerenciamento de E/S
OBSOLETA

E/S Programada

CPU verifica repetidamente cada dispositivo. Muito ineficiente — consome todos os ciclos da CPU com verificações.

→
LEGADA

E/S por Interrupção

Dispositivos notificam a CPU quando prontos. Libera a CPU, mas ainda exige sua intervenção direta.

→
ATUAL

DMA

Controlador dedicado gerencia transferências. CPU fica livre para processar. Máxima eficiência.

Aula 4 · E/S Programada e por Interrupção
E/S Programada vs Interrupção

E/S Programada (Polling)

Verificação contínua

  • CPU executa um loop verificando o status do dispositivo
  • Enquanto espera, não faz mais nada (busy waiting)
  • Simples de implementar
  • Desperdiça ciclos de CPU
  • Adequada apenas para sistemas simples e embarcados
VS

E/S por Interrupção

Notificação por hardware

  • Dispositivo envia um sinal (IRQ) quando pronto
  • CPU pode executar outras tarefas enquanto aguarda
  • Mais complexa — exige tratador de interrupções
  • Muito mais eficiente que polling
  • Usada em todos os sistemas operacionais modernos

Evolução: E/S Programada → E/S por Interrupção → DMA → Canais de E/S

Aula 4 · DMA
DMA — Acesso Direto à Memória

O DMA permite que dispositivos de E/S transfiram dados diretamente para/da memória, sem intervenção constante da CPU.

Um controlador de DMA assume a gestão da transferência, liberando o processador para outras tarefas.

Aula 4 · DMA — Funcionamento
DMA — Passo a Passo
PASSO 1

Solicitação

CPU programa o controlador DMA com endereço, quantidade de dados e direção da transferência.

→
PASSO 2

Arbitragem

DMA solicita acesso ao barramento. Quando concedido, assume o controle da transferência.

→
PASSO 3

Transferência

Dados fluem diretamente entre dispositivo e memória, sem passar pela CPU.

→
PASSO 4

Interrupção

Ao concluir, o DMA envia uma interrupção à CPU informando que a operação terminou.

⚠ Desafios do DMA

Sincronização com a CPU pelo barramento, segurança de acesso à memória e coerência de cache.

✓ Benefício Principal

CPU fica livre para executar outras tarefas durante toda a transferência de dados.

Aula 4 · Canais de E/S
Canais de Processadores de E/S

Canais de E/S são circuitos dedicados que operam independentemente da CPU, permitindo múltiplas operações de E/S simultâneas.

Canal Seletor (Simples)

Conecta um dispositivo específico por vez. Otimizado para transferências de alta velocidade.

Canal Multiplexador

Gerencia múltiplos dispositivos simultaneamente, alternando entre eles conforme necessidade.

FUNCIONAMENTO

1 Inicialização — Configura o dispositivo
2 Transferência — Move dados via buffers
3 Controle — Monitora estado e fluxo
4 Interrupção — Notifica a CPU ao concluir
Encerramento · Estudo de Caso
Estudo de Caso: Biblioteca Central

A Biblioteca Central de São Paulo enfrenta lentidão e falhas no sistema. Equipamentos antigos não suportam o volume de dados. A equipe de TI precisa modernizar a infraestrutura.

1. CPU

Processador sobrecarregado, ciclos maximizados, registradores no limite.

2. Memória

RAM sempre cheia, alto índice de cache misses, acesso excessivo ao disco.

3. Mem. Secundária

Discos magnéticos lentos, mídias de backup obsoletas.

4. E/S

Dispositivos antigos, sem DMA, canais de E/S saturados.

SOLUÇÃO PROPOSTA

✓ Processador multi-core de alta frequência
✓ Expansão de RAM + cache multinível
✓ Substituição de HDD por SSD + NAS/SAN
✓ Interfaces com DMA + USB 3.x / Thunderbolt
